# prompt-rules.md — Claude Code 作業ルール

このリポジトリで Claude Code（および任意のAIアシスタント）が作業するときのルール。
**作業を始める前に必ずこのファイルと [README.md](../README.md) を読むこと。**

---

## 作業開始時のチェックリスト

1. **README を読む** — [README.md](../README.md) で全体構成と方針を把握する。
2. **対象アプリの場所を確認する** — [PROJECTS.md](../PROJECTS.md) で `apps/app-xxx` のどれかを特定する。
3. **対象アプリの README を読む** — `apps/app-xxx/README.md` で固有仕様を確認する。
4. **作業範囲を1アプリ（または明示された範囲）に限定する。**

---

## やってよいこと

- 指定されたアプリ・ファイルの実装・修正・改善。
- 変更後の `npm run build` 等による動作確認。
- ドキュメントの更新（変更内容に合わせて README / PROJECTS を最新化）。

---

## やってはいけないこと（厳守）

- ⛔ **関係ないアプリのファイルを変更しない。** 指定された `apps/app-xxx` の外に触れない。
- ⛔ **勝手に大規模変更をしない。** フォルダ移動・リネーム・一括置換・依存の大規模更新は事前提案・承認制。
- ⛔ **削除は特に慎重に。** 削除前に対象を確認し、理由を説明し、退避（バックアップ）を先に行う。
- ⛔ **秘密情報に触れない・コミットしない。** `.env` / `.env.local` / APIキー / トークンは読み書き・出力しない。
- ⛔ **Vercel設定・本番デプロイを勝手に変更しない。** 設定変更はオーナーが手動で行う。
- ⛔ **承認なしに push しない。** push は本番デプロイを誘発しうるため、必ず事前確認する。

---

## 変更前に必ず説明すること

何かを変更する前に、以下を簡潔に伝える:

1. **何を** 変更するか（対象ファイル・フォルダ）
2. **なぜ** 変更するか
3. **影響範囲**（他アプリ・本番への影響の有無）
4. 大規模・不可逆な操作なら **承認を求める**

---

## 変更後に提示すること

- 変更・作成したファイルの **絶対パスまたはルート相対パス**。
- 動作確認に使うコマンド:

```bash
cd apps/app-xxx
npm install     # 依存が変わった場合のみ
npm run build   # 本番ビルドが通るか
npm run dev     # http://localhost:3000 で表示確認
```

静的アプリ（app-elsa-like）:
```bash
cd apps/app-elsa-like && open index.html   # または npx serve .
```

---

## データ保護の原則

- 不可逆な操作（削除・`.git`操作・一括移動）の前に、必ずバックアップまたはリモートへの退避状況を確認する。
- 各アプリは個別 GitHub リポジトリにも履歴がある。迷ったら破壊的操作を止め、オーナーに確認する。
