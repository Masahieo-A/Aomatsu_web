# 出力仕様（カード化・表示形式の管理）

このドキュメントは、**探究ファシリテーターAI Webアプリ**において、AIの返答がどのように「指示」され、どこで「カード化」などの表示形式が決まるかを説明する。

## 結論（質問への答え）

| 観点 | どこで管理しているか |
|------|----------------------|
| 対話の目的・STEP1〜4の流れ・ペルソナ・マトリクス表をMarkdownで出す等 | **Gemini API の `system_instruction`**（`lib/systemInstruction.ts` の本文 + **`JSON_OUTPUT_APPENDIX`**） |
| **1ターン分の応答の機械可読な形** | **Gemini `generationConfig.responseMimeType: application/json` + `responseSchema`**（`lib/assistantResponseSchema.ts`） |
| カードグリッド・レンズ・視点セクション等へのマッピング・`*` 除去 | **Next.js**（`lib/parseGeminiResponse.ts` が **JSON を優先パース**し、失敗時のみ従来のヒューリスティックへフォールバック） |
| Python 等の別バックエンド処理 | **本プロジェクトでは使用していない** |

**システムプロンプト**は会話内容と「JSON ブロックにどう分けるか」を指示し、**API の responseSchema** が出力の形を **一意に拘束**する。表示は **`ParsedBlock`（`types/index.ts`）** に落とし込んでから React コンポーネントが描画する。

---

## 1. Gemini API（システムプロンプト）が担うこと

- **ファイル**: `lib/systemInstruction.ts`
  - 要件ベースの本文: `SYSTEM_INSTRUCTION`（`要件定義.md` と整合）
  - API 用の結合: `SYSTEM_INSTRUCTION_FOR_API` = 上記 + `JSON_OUTPUT_APPENDIX`（JSON ブロックの使い分けルール）
- **渡し方**: `lib/gemini.ts` で `getGenerativeModel({ systemInstruction: SYSTEM_INSTRUCTION_FOR_API, generationConfig: { responseMimeType: "application/json", responseSchema: GEMINI_ASSISTANT_RESPONSE_SCHEMA } })`

主に次を指示している。

- 探究マトリクスのワークフロー（STEP1〜4）
- ペルソナ・日本語・ガードレール
- STEP3 のマトリクスは **markdown ブロックの `content` 内**に Markdown 表として含める
- **付録 JSON_OUTPUT_APPENDIX**: `version` / `blocks` / `kind`（`markdown` | `options` | `lensPicker` | `sections` | …）の使い方

**API 側**: `responseSchema`（`lib/assistantResponseSchema.ts`）が **スキーマとして** UI 相当の構造を定義する（プロンプトだけに頼らない）。

---

## 1.1 Gemini→Next.js の受け渡し形式（JSON）

### 実装済みの内容

| 項目 | 内容 |
|------|------|
| **Gemini の応答本文** | **`application/json`**（`responseSchema` 準拠の 1 オブジェクト）。`lib/gemini.ts` の `generateGeminiAssistantJson` |
| **ストリーミング** | **使わない**（1ターン分の JSON が揃ってから `text/plain` でクライアントへ返す） |
| **要件定義書**（`要件定義.md`） | 旧版は **JSON スキーマの記載なし**。コード側は **`SYSTEM_INSTRUCTION_FOR_API` + `GEMINI_ASSISTANT_RESPONSE_SCHEMA`** で実体を持つ |
| **フォールバック** | `tryParseAssistantJsonToBlocks` が失敗したときだけ、従来の **プレーンテキスト・ヒューリスティック**（`parseGeminiResponse` 後段） |

### 要件定義書との関係

`要件定義.md` のシステムプロンプト節は **人間向けワークフロー**のソース。実装では **JSON 付録**と **API スキーマ**が「ブロック種別とフィールド」を上書き定義する。

### メンテナンス時

スキーマや `kind` を変えた場合は **`assistantResponseSchema.ts`（API）** と **`tryParseAssistantJsonToBlocks`（検証）** と **`JSON_OUTPUT_APPENDIX`（プロンプト）** を揃える。

## 2. Next.js（フロント／API Route）が担うこと

### 2.1 API ルート（サーバー）

- **ファイル**: `app/api/chat/route.ts`
- **役割**: 会話履歴を受け取り、`generateGeminiAssistantJson` で **JSON 文字列**を取得し、そのまま本文として返す（構造解析はクライアント側）。

### 2.2 返答テキストの解析（カード化の中核）

- **ファイル**: `lib/assistantResponseSchema.ts` の `tryParseAssistantJsonToBlocks` → 失敗時は `lib/parseGeminiResponse.ts` のヒューリスティック
- **通常経路**: モデル出力は **JSON**。`version` + `blocks[]` の各 `kind` を **`ParsedBlock`** に変換し、一意に UI へ対応させる。
- **フォールバック経路**: JSON パース不能・検証失敗時のみ、従来どおり **自由形式テキスト**として解析するため、分類は **一意に限らない**。

  - `options` … **カードグリッド**（`OptionGrid`）
  - `lensPicker` … **レンズ専用 UI**（`LensPicker`）
  - `sections` … **視点カード**（`PerspectiveCards`）
  - `steps` / `themes` … 各カードコンポーネント
  - `markdown` … `react-markdown` で表示（表・段落はここに含める）

- **付随処理**:
  - `*` の削除（表示方針として UI 側で実施）
  - 「福祉（例：…）」のような表記から **カード用ラベル**と **tooltip 用の原文**を分ける、など

### 2.3 表示コンポーネント

- **例**: `components/OptionGrid.tsx`, `LensPicker.tsx`, `PerspectiveCards.tsx`, `MessageBubble.tsx` など
- **役割**: `parseGeminiResponse` の結果に応じて、実際のカード・チップ・Markdown を描画する。

### 2.4 初回画面だけの固定文

- **ファイル**: `components/ChatWindow.tsx` の `INITIAL_ASSISTANT_MESSAGE`
- **役割**: **API と同じ JSON 封筒**（`JSON.stringify({ version: 1, blocks: [...] })`）で渡す。`markdown` ブロック + `options` ブロック（9 分野）。

---

## 3. Python は使っているか

**いいえ。** 本リポジトリの実装は **Next.js（TypeScript）+ Gemini API** のみであり、出力形式の「噛ませ処理」は **Node 上の API Route とブラウザ上の TS** で行っている。

## 4. 変更したいときの指針

| やりたいこと | 触る場所の目安 |
|--------------|----------------|
| AIの話し方・STEP の進め方・表を出す指示 | `SYSTEM_INSTRUCTION`（要件定義と整合） |
| **ブロック種別・JSON フィールドの契約** | **`GEMINI_ASSISTANT_RESPONSE_SCHEMA` + `JSON_OUTPUT_APPENDIX` + `tryParseAssistantJsonToBlocks`**（三つを揃える） |
| フォールバック時のテキスト解析 | `lib/parseGeminiResponse.ts`（ヒューリスティック部分） |
| カードの見た目・レンズ UI のレイアウト | `components/*` |
| 初回メッセージの文言・9 分野の出し方 | `components/ChatWindow.tsx`（JSON 封筒） |

**注意**: 通常は **API の responseSchema** が効くため、ブロック構造を変えるときは **プロンプト付録・スキーマ・パーサ**を同時に更新する。フォールバック経路に頼ると **一意性は保証されない**。

## 6. 交差点まとめ（`/intersection`）

- **レンズタップ時**はチャット API を呼ばず `sessionStorage` に会話を退避して遷移する（`components/ChatWindow.tsx`）。
- **要約・問い例3つ**は `POST /api/intersection-summary` → `generateIntersectionSummaryJson`（`INTERSECTION_SUMMARY_SCHEMA`）。
- **締めの一文**は Gemini ではなく **`INTERSECTION_CLOSING_COMMENT`（コード定数）** で固定表示（`app/intersection/page.tsx`）。
