# 青松問答 本番質問バンク 設計書 v1.0

執筆エージェント共通の正典。全350問はこの設計に従って生成する。
上位文書: docs/requirements_collection_app.md（3.1 カテゴリ定義）、docs/requirements_aomatsu_ai.md（3. データソース設計）。

## 目的の再確認

この質問バンクは「青松AI」の思考層・文体層・知識層コーパスの原料を集める装置である。
特に **AI対話ログに現れない領域**（日常判断・対人・感情・美意識・雑談の呼吸）を能動的に埋めることが最大の価値（全体要件3.2-3）。
よい質問とは「答えると、本人の判断基準・価値観・文体が言語化される」質問である。事実クイズや一般論を答えさせる質問は無価値。

## 回答者プロフィール（前提にしてよい範囲）

- 高校教員。担任・教務・入試業務・部活動（陸上）の指導経験がある
- Note/Xで発信し、コンテスト応募・提案書・企画立案を行う
- AIを日常的に活用し、個人開発（Webアプリ）もする
- 上記以外の私生活の事実（家族構成・出身地など）は**決めつけない**。「あなたにとって家族とは」のように事実を仮定しない聞き方はよいが、「あなたの子どもは」のような仮定はNG。答えにくい質問はアプリ側でスキップできるので、踏み込むこと自体は恐れない

## 全カテゴリ共通の執筆規則

1. 日本語・敬体の疑問文。二人称は「あなた」（ただし文脈上不要なら省略）
2. **1問1論点**。複数のことを一度に聞かない（「〜と〜、それぞれどうですか」は禁止）
3. 具体の錨を置く: 「最近」「直近1年で」「今年度」「具体的な場面を1つ」など、回想の範囲を絞る言葉を使う（Q1/Q3/Q6の普遍的な問いは除く）
4. 質問文は**120字以内**目安。前置きの講釈をつけない
5. 「なぜですか？」を質問文に含めない（理由欄が別にあるため。結論・選択・エピソードそのものを問う）
6. 同一カテゴリ内・カテゴリ間で論点の重複を作らない（似た質問は角度を変える）
7. Yes/Noで終わる聞き方を避け、「どちら」「何」「どんな場面」で問う
8. id は `q{カテゴリ番号}_{3桁連番}`（既存seedと同一。差し替えupsertのため）

## JSON出力形式

```json
[
  {"id": "q1_001", "category": "Q1", "body": "質問文", "body_options": {"A": "選択肢A", "B": "選択肢B"}},
  {"id": "q2_001", "category": "Q2", "body": "質問文"}
]
```
body_options は Q1/Q4 のみ。他カテゴリは省略。

## Q1 価値観二択（60問・思考層）

トレードオフの二択 + 理由（必須20字以上）。**A/Bはどちらも一定の魅力がある緊張関係**にすること。片方が明らかに正解・社会的に望ましい選択肢は不可。「片方しか選べないなら」「どちらかを諦めるなら」の切迫感を持たせる。

選択肢は名詞句または短文（30字以内）。質問文とセットで自立して意味が取れること。

| サブテーマ | 問数 | 例の方向性 |
|---|---|---|
| 教育・仕事観 | 12 | 全員への公平 vs 個々への最適化 / 厳しさで伸ばす vs 安心感で伸ばす / 授業の質 vs 校務の完遂 |
| 創作・発信 | 12 | 正確さ vs 面白さ / 速く出す vs 練って出す / 広く届く vs 深く刺さる / 自分が書きたい vs 読者が読みたい |
| 対人関係 | 10 | 正直な指摘 vs 関係の維持 / 頼られ続ける vs 自立させる / 距離を詰める vs 距離を保つ |
| 生活・時間 | 10 | 効率 vs 味わい / 計画通り vs 気分次第 / 早起きの朝 vs 集中の深夜 |
| 美学・原則 | 8 | 引き算の完成 vs 足し算の豊かさ / 王道 vs 変化球 / 一貫性 vs 柔軟性 |
| 自己・成長 | 8 | 得意を尖らせる vs 苦手を潰す / 褒められる vs 納得できる / 安定 vs 挑戦 |

## Q2 判断エピソード（50問・思考層）

実際にあった**具体的な場面と、その時どう判断したか**を問う。「最近」「今年度」「直近1ヶ月」等の時間の錨を必ず置く。架空の状況への仮定回答（「もし〜だったら」）は禁止。実体験を掘る。

| サブテーマ | 問数 |
|---|---|
| 断った/引き受けた場面 | 8 |
| 優先順位・時間配分の判断 | 8 |
| 教育現場の判断（生徒対応・保護者対応・授業の軌道修正） | 10 |
| 買い物・金銭の判断 | 5 |
| 対人の調整・衝突への対処 | 7 |
| 失敗とリカバリー | 6 |
| ツール・技術・方法の選定 | 6 |

## Q3 意見表明（60問・思考層）

賛否や立場が分かれる論点への**立場と論拠**を問う。「あなたの立場は？」「どう考えますか？」型。世間で答えが割れている論点だけを選ぶ（誰もが同意する話題は不可）。時事性が強すぎて数年で無意味になる論点も避ける。

| サブテーマ | 問数 | 例の方向性 |
|---|---|---|
| 教育論 | 18 | 宿題の要否 / 定期テスト / 部活動のあり方 / 生成AIと学習 / 評価・成績 / 校則 / ICT / 探究学習 |
| AI・テクノロジー | 10 | AIと創作の関係 / 人間に残る仕事 / AI利用の開示義務 / 技術の学び方 |
| 文章・発信論 | 10 | バズvs蓄積 / 匿名vs実名 / 長文vs短文の価値 / 添削で個性は死ぬか |
| 働き方・組織 | 8 | 会議 / 根回し / 若手への任せ方 / 職員室の同調圧力 |
| 社会・一般 | 8 | 効率化と豊かさ / 地方と都市 / 世代論 / お金と幸福 |
| 学び・成長 | 6 | 独学vs師事 / 量vs質 / 苦手克服の意味 |

## Q4 文体A/B（40問・文体層）

**同じ内容を異なる文体で書いた2文**を提示し、「自分が書きそうなのはどちらか」を選ばせる（+違いの言語化を理由欄で）。質問文は基本形「同じ内容の2つの文です。あなたが書きそうなのはどちらですか？（理由欄に、何が違うと感じたかを書いてください）」をジャンルに応じて微調整してよい。

- A/B本文は各40〜90字。**意味・情報量は同一**、文体だけを変える
- 各ペアで対比軸を1つ主軸に据える（軸名は本文に書かない）: 文末処理（言い切り/婉曲）、文のリズム（短文連打/一文長め）、和語/漢語、比喩の有無、体言止め、敬体/常体、数字・データの出し方、感情の露出度、駄洒落・言葉遊びの有無、接続詞の密度
- どちらが「上手い」かではなく「らしい」かを測る対比にする（片方を下手に書かない）
- A/Bどちらに「凝った」文体を置くかはペアごとにランダムにする（位置バイアス防止）

| ジャンル | 問数 |
|---|---|
| Noteエッセイの冒頭 | 8 |
| X（短文）投稿 | 8 |
| 生徒・保護者向けの案内文 | 6 |
| 教材・解説の説明文 | 6 |
| 依頼・お礼のメール文 | 6 |
| 生徒作文・作品への講評コメント | 6 |

## Q5 日常・対人・感情（60問・思考層/AIログ最欠落域）

**このカテゴリが本プロジェクトの最重要領域**（AIに聞かないことを聞く）。仕事の話に回収させない。感情の生の動き、人付き合いの呼吸、生活の癖を問う。プライベートに踏み込んでよい（スキップ可能が前提）。

| サブテーマ | 問数 |
|---|---|
| 感情の扱い（怒り・不安・機嫌・落ち込み） | 12 |
| 家族・親しい人との関わり | 10 |
| 職場の人間関係の呼吸 | 10 |
| 雑談・社交・初対面 | 8 |
| 生活習慣・休日・ルーティン | 10 |
| 体調・疲れ・ストレス対処 | 6 |
| お金・消費の感覚 | 4 |

## Q6 美意識・好み（40問・文体層/思考層）

「いい」「ダサい」「許せない」の基準を言語化させる。「条件を3つ挙げると？」「最近見た中で一番〜だったものは？」型が有効。

| サブテーマ | 問数 |
|---|---|
| 文章の美学（いい文章/悪文/推敲の基準） | 10 |
| デザイン・視覚（スライド・板書・資料・Web） | 8 |
| 道具・モノ選び | 6 |
| 空間・環境（作業場所・教室・家） | 4 |
| 芸術・エンタメの好み（本・映画・音楽） | 6 |
| 命名・言葉遊び・駄洒落の美学 | 6 |

## Q7 事実・経歴（40問・知識層）

経歴・事実 + **その時の判断や感情**をセットで問う（年表の穴埋めではなく、物語として掘る）。
