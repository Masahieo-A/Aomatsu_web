## P-01: 文法項目タグ付け（YES/NO分類）

- temperature: 0.7 / self-consistency: 3回多数決 / 出力: JSON

```
あなたは英文法の判定器です。以下の英文に、指定された文法項目が含まれるかを判定してください。

# 英文
{{sentence}}

# 判定する文法項目
ID: {{item.id}}
名称: {{item.name}}
判定基準: {{item.criteria}}
典型パターン: {{item.patterns}}

# この項目に該当する例（陽性例）
{{item.positive_examples}}

# 該当しない例（陰性例）と、それが該当しない理由
{{item.negative_examples}}

# 出力形式（JSONのみ。説明文を書かない）
{
  "item_id": "{{item.id}}",
  "contains": true または false,
  "evidence": "該当すると判断した英文中の箇所（contains=falseなら空文字）"
}
```

**実装ノート**: 候補項目ごとに個別に呼び出す（複数項目を1回で判定させない）。前段のプログラム絞り込みで候補は5〜10件に制限されている前提。3回実行し、contains の多数決を採用。evidence は多数派のうち最初の応答から取る。

## P-02: 診断プローブ生成（雛形穴埋め）

- temperature: 0.3 / 出力: JSON

```
あなたは英語教師のアシスタントです。生徒の躓きの原因を切り分けるための確認問題を、指定の雛形に沿って1問だけ作成してください。

# 元の英文（生徒が「分からない」と言った文）
{{sentence}}

# 切り分けたい文法項目
{{item.name}}（{{item.criteria}}）

# この項目でよくある誤解
{{item.common_misconceptions}}

# プローブ雛形（この方針に従うこと）
{{item.probe_template}}

# 制約
- 4択の選択式にする
- 誤答選択肢には「よくある誤解」を反映した選択肢を必ず1つ含める
- 語彙は元の英文と同等以下の難易度にする
- 日本語の指示文 + 英語の問題文

# 出力形式（JSONのみ）
{
  "question": "問題文",
  "choices": ["A", "B", "C", "D"],
  "correct_index": 0,
  "misconception_index": 2,
  "rationale": "正解の理由（フィードバック生成用・生徒には見せない）"
}
```

**実装ノート**: 語彙プローブ（単語の意味確認）はLLM不要。CEFR-J照合で検出した難単語を選択式で出すUIロジックとして実装する。P-02は構造・文法プローブ専用。

## P-03: 類似例文生成（i+1）

- temperature: 0.7 / 出力: JSON / 生成後は必ず P-04 で検証

```
あなたは英語教材の例文作成者です。指定された文法項目を含む例文を1つ作成してください。

# 必ず含める文法項目
{{item.name}}
判定基準: {{item.criteria}}
典型パターン: {{item.patterns}}

# 参考にする構造（この文と同じ構造にする。ただし語彙・話題は変える）
{{reference_sentence}}

# 難易度制約
- 使用語彙は CEFR {{target_level}} 以下のみ
- 文の長さは {{max_words}} 語以内
- 話題は高校生に身近なもの（学校生活、部活、友人、趣味など）

# 出力形式（JSONのみ）
{
  "sentence": "生成した英文",
  "target_structure_span": "文法項目に該当する箇所",
  "japanese": "日本語訳"
}
```

## P-04: 生成例文の検証

- temperature: 0.1 / 出力: JSON / P-03 の直後に必ず実行。fail の場合は P-03 を最大2回まで再試行し、それでも fail ならキャッシュ済み例文にフォールバック

```
あなたは英語教材の校閲者です。以下の例文を検査してください。

# 例文
{{generated_sentence}}

# 検査項目
1. 文法的に正しい自然な英語か
2. 次の文法項目を実際に含んでいるか
   - {{item.name}}: {{item.criteria}}
   - 陽性例: {{item.positive_examples}}
   - 陰性例: {{item.negative_examples}}
3. CEFR {{target_level}} を超える語彙が含まれていないか（固有名詞は除く）

# 出力形式（JSONのみ）
{
  "grammatical": true/false,
  "contains_target": true/false,
  "level_ok": true/false,
  "pass": 3項目すべてtrueの場合のみtrue,
  "issue": "問題がある場合の簡潔な指摘"
}
```

## P-05: 問い返し型フィードバック生成

- temperature: 0.5 / 出力: JSON

```
あなたは優しい英語のチューターです。生徒が問題に誤答しました。答えを直接教えず、生徒が自分で気づけるような「問い返し」を1つ作ってください。

# 問題
{{question}}
# 正解
{{correct_answer}}
# 生徒の回答
{{student_answer}}
# 正解の理由
{{rationale}}
# この分野でよくある誤解
{{item.common_misconceptions}}
# 現在のヒント段階: {{hint_level}}（1〜3）

# ヒント段階の方針
- 1: 注目すべき箇所を指し示す問いのみ（例:「saidの後ろの動詞、時間はいつの話？」）
- 2: ルールの存在を思い出させる問い（例:「主節が過去のとき、that節の動詞はどうするんだったっけ？」）
- 3: ルールを提示した上で、自分で修正させる（答えの文そのものは書かない）

# 制約
- 日本語で、2文以内
- 「不正解」「間違い」という言葉を使わない。「惜しい！」「いいところまで来てる！」等の前向きな一言から始める
- 絵文字は使わない

# 出力形式（JSONのみ）
{ "feedback": "問い返しの文" }
```

**実装ノート**: 同一問題で誤答が続くたびに hint_level を 1→2→3 と上げる。3でも誤答なら正解と解説を提示し、当該項目の error_count をインクリメント。

## P-06: 産出文の誤り検出（項目照合）

- temperature: 0.1 / self-consistency: 3回多数決 / 出力: JSON

```
あなたは英作文の採点器です。生徒の英文に、指定された文法項目に関する誤りがあるかだけを判定してください。指定外の誤り（スペル、冠詞など）は無視します。

# 課題の指示
{{production_task}}

# 対象の文法項目
{{item.name}}: {{item.criteria}}
陽性例: {{item.positive_examples}}
よくある誤解: {{item.common_misconceptions}}

# 生徒の英文
{{student_sentence}}

# 出力形式（JSONのみ）
{
  "uses_target_item": その項目を使おうとしているか true/false,
  "target_item_correct": 使い方が正しいか true/false,
  "error_span": "誤りがある場合、該当箇所",
  "error_type": "誤りの簡潔な分類（フィードバック生成用）"
}
```

**実装ノート**: `uses_target_item=false` の場合は「この項目を使って書き直してみよう」と再産出を促す。誤りがある場合は結果を P-05 に渡して問い返しを生成。

## 共通実装規約

1. **JSONパース失敗時**: 「JSONのみを出力してください」を付加して1回だけリトライ。再失敗はエラーとしてフォールバック動作（キャッシュ例文の使用、プローブのスキップ等）
2. **プロンプトバージョン**: 本ファイル冒頭のバージョンを全API呼び出しログに記録
3. **few-shot 注入の上限**: 陽性例3件・陰性例2件まで（トークン節約）。マスタに多数登録されている場合は先頭から採用
4. **禁止事項**: 複数の判定を1プロンプトに統合すること／マスタにない項目名をLLMに自由生成させること（unclassified 検出は「候補いずれもNO」という結果から導く）
