# 構成メモ（開発者向け）

- **GitHubリポジトリ**: 未設定（ローカルにgitリポジトリはあるが `origin` リモート未設定。GitHub/Vercelの作成・Importはご自身で行う運用）
- **フレームワーク / 構成**: npm workspaces によるモノレポ（`zdp-monorepo`）。Next.js 14 + TypeScript。
  - `apps/zdp` — 生徒向け診断アプリ（Next.js、開発時 port 3000）
  - `apps/master-editor` — マスタ管理アプリ（Next.js、開発時 port 3100）
  - `packages/prompts` — 両アプリ共有のプロンプトテンプレート（`templates/*.md` → `npm run sync-prompts` で生成）
  - `packages/schema` — `grammar_master.json` のスキーマ定義・検証（DAG・参照整合性）
  - ルート `package.json` の主なスクリプト: `dev:zdp` / `dev:master` / `build:zdp` / `build:master` / `sync-prompts`
- **DB / データ管理**: ZDPアプリはGoogleスプレッドシート（未設定時はローカルJSON `apps/zdp/.data/*.json` にフォールバック）。マスタアプリは `apps/master-editor/.data/master_working.json` が正データ（`MASTER_DATA_DIR` でGoogle Drive同期フォルダ等に変更可）。
- **APIキー管理**: ルート `.env.example` を各アプリ直下に `.env.local` としてコピーして使用。キー名: `GEMINI_API_KEY`, `GEMINI_MODEL`, `GOOGLE_SHEETS_ID`, `GOOGLE_SERVICE_ACCOUNT_EMAIL`, `GOOGLE_PRIVATE_KEY`, `MASTER_BASIC_USER`, `MASTER_BASIC_PASS`, `MASTER_DATA_DIR`（値は本ファイルに記載しない）。
- **デプロイ先**: 未設定（`.vercel` ディレクトリなし）。README の想定はVercelで、2アプリを別プロジェクト（Root Directory: `apps/zdp` / `apps/master-editor`）としてデプロイする構成。GitHub/Vercelの作成・Importは開発者本人が手動で行い、pushはSSHのみで運用。
