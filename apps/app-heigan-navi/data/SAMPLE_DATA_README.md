# exam-methods.json について

このファイルは `併願照合アプリ_要件定義.md` のスキーマ（§4 ExamMethod / SubjectRequirement / SubjectCode / Evidence）に沿って、
`juken-data/` 配下の実際の入試要項テキスト（.txt、PDFの逐語抽出）からClaudeエージェントが直接読み取って作成した**サンプルデータ**です（2026-07-07時点、13大学・798件、うち793件が機械検証OKで`confirmed`）。

- 本来のパイプライン（§5 抽出パイプライン：Gemini構造化＋原文一致検証）は通していません。エージェントが手動で`.txt`を読み、逐語引用（evidence.quote）を書き写しています。
- **機械検証は実施済み**：全evidence.quoteを対応する`.txt`ソースファイルと文字列一致検証（NFKC・空白正規化後の部分一致、§5.3のverify()ロジックを踏襲）。当初の抽出パスで不一致だった分は修復パスで原文と突き合わせて引用・値を修正済みです。
- 修復パスでは引用の直しだけでなく**値の実質的な誤りも複数発見・修正**しました（名城農学部の「化学必須」→理科選択、愛知学院の事前体験型/事前課題型の取り違え4件、中京国際思考型の試験日、金城共テ前期の選択科目数）。この種の誤りは機械検証だけでは捕捉できないため、教員レビューの重要性は変わりません。
- `heiganPolicy`（専願/併願可の判定）は、一度「引用文の意味と逆の値を設定する」バグが実際に見つかっているため、ヒューリスティックな矛盾チェックも別途実施済みです。
- `reviewedBy: "サンプルデータ(自動生成・全件版)"`は仮の値であり、実際の教員確認（§6 教員確定ツール）を経たものではありません。
- そのため **このデータをそのまま生徒への進路指導・出願判断に使用しないでください**。面談等で使う前に、必ず教員が`evidence.pdfFile`/`evidence.page`を辿って元のPDF（`data/source-links.json`経由で大学公式サイトの一次資料にリンク）と突き合わせ、内容を確認・修正した上で`reviewStatus`を確定させてください。
