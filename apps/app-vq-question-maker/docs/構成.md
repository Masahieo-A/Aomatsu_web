# 構成メモ（開発者向け）

- **GitHubリポジトリ**: このローカルディレクトリは git 管理下（ブランチ `main`、コミット1件「Initial commit」）ですが、`git remote -v` の結果は空で、リモートは未設定です。なお GitHub 上には同名の公開リポジトリ `Masahieo-A/VQ_like_Question_Maker` (https://github.com/Masahieo-A/VQ_like_Question_Maker) が存在することを確認しましたが、このローカルコピーからは紐付いていません（別のローカルコピー・パスからpushされた可能性があります。`.claude/settings.local.json` の履歴に `/tmp/repo-dash` 等の別パスでの push 作業の形跡があります）。
- **フレームワーク / 構成**:
  - backend: Python / FastAPI（`backend/requirements.txt`: fastapi, uvicorn, pillow, numpy, pymupdf, pytesseract, python-docx, openpyxl）。実体は `backend/app/main.py` の単一ファイルで、PDF OCR、Word/Excel エクスポート用の API。
  - frontend: Next.js 14.2.4 + React 18.3.1 + TypeScript 5.4.5（`frontend/package.json`）。`frontend/app/`, `frontend/lib/insightDataset.ts`, `frontend/data/vision-quest-insight.json` を利用。
- **DB / データ管理**: 専用DBサーバーはなし。教材データはルート直下の CSV「スキャン_文字起こしDB - １ (1).csv」を元に生成したと思われる `frontend/data/vision-quest-insight.json` を、アプリがファイルとして読み込んで利用（`frontend/lib/insightDataset.ts` 経由）。
- **APIキー管理**: リポジトリ内に `.env` 系ファイルは見当たりません（`find . -iname "*.env*"` で確認、frontend/backend双方とも該当なし）。`.gitignore` に `.env` / `.env.*` の除外設定あり。README記載によると `OPENAI_API_KEY` と `OPENAI_MODEL` を Vercel の環境変数に設定して使う想定（キー名のみ、値はどこにも記載なし）。
- **デプロイ先**: 未デプロイ（このローカルコピーには `.vercel/` ディレクトリ・`vercel.json`・CI/CDワークフローのいずれも見当たりません。node_modules内のサードパーティ製ワークフローファイルを除く）。
- **関連ドキュメント**: 機能要件は docs/要件定義.md を参照。
