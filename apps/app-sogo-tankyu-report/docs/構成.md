# 構成メモ（開発者向け）

- **GitHubリポジトリ**: `git@github.com:Masahieo-A/sogo-tankyu-report.git`
- **フレームワーク / 構成**: 素のHTML/CSS/JS（フレームワークなし）の静的サイト。データ更新は Google Apps Script（`apps-script/`）が Classroom 提出物取得 → PDF変換 → `data/schedule.json` 生成 → GitHubへの直接コミットまでを自動化し、Vercelが自動デプロイする構成。
- **DB / データ管理**: 専用DBなし。Google スプレッドシート「グループ一覧」シートがマスタデータ。Apps Scriptがシートを読み取り `data/schedule.json`（静的JSON）を生成してサイトに反映する。
- **APIキー管理**: リポジトリ内に `.env` 系ファイルなし。GitHubトークンはスプレッドシートのメニュー「🔑 GitHubトークンを設定」からApps Scriptの スクリプトプロパティに保存する運用（シートのセルには保存しない）。
- **デプロイ先**: Vercel。GitHubへのpushをトリガーに自動デプロイ（`.vercel/project.json` は未生成、Vercel側でGit連携設定済みの想定）。`vercel.json` にセキュリティヘッダー（CSP等）を定義。ローカル確認は `npx serve -l 8080`（`serve.json` で `cleanUrls: false`）。
