# 口頭試問プリント生成ツール（app-oral-exam-generator）

> **English Hub ポータル内アプリ**（`aomatsu-web` monorepo / `apps/app-oral-exam-generator`）。教員向けユーティリティです。
> 生徒の英文を扱う都合上、**先生のパソコンの中だけでローカル起動して使う設計**で、Vercel などへは公開デプロイしていません。トンマナはポータル共通の `docs/design-system.md`（緑基調）に準拠しています。

生徒の英作文（1クラス数十人分）を貼り付けると、**各生徒の英文の内容に基づく個別の追随質問（2〜3問）・模範解答・採点基準**を自動生成し、A4小テストプリントとして印刷できるツールです。先生のパソコンの中だけで動くので、生徒の英文が外部のサーバーに保存されることはありません（AI生成のために Google の Gemini API に送信されるのみです）。

> **開発状況**：Phase 3（全フェーズ）まで実装済みです。英文の投入 → AI問題生成 → 3段階の自動検証 → 教員レビュー → A4プリント印刷 → 採点入力・分析まで一通り使えます。自動検証を通過した問いでも、**印刷前のレビュー確認（全問承認）は必須工程**です。

---

## 1. 使い始める前の準備（初回のみ）

### 1-1. Node.js のインストール

このツールを動かすには「Node.js」（バージョン20以上）というソフトが必要です。

1. https://nodejs.org/ja にアクセス
2. 「LTS（推奨版）」と書かれたボタンからダウンロードしてインストール
3. インストール確認：「ターミナル」（Macの場合）を開いて `node --version` と入力し、`v20` 以上の数字が出ればOK

### 1-2. Gemini API キーの発行（無料）

AI(Gemini)を使うための「鍵」を発行します。クレジットカード登録は不要で、無料枠の範囲で使えます。

1. https://aistudio.google.com/ にアクセスし、Google アカウントでログイン
2. 左メニューまたは画面上部の「**Get API key**（APIキーを取得）」をクリック
3. 「**APIキーを作成**」をクリックし、表示された長い文字列（例：`AIza...`）をコピー

### 1-3. 設定ファイルの作成

1. このフォルダにある `.env.local.example` をコピーして、名前を `.env.local` に変更
2. テキストエディタで開き、`GEMINI_API_KEY=` の後ろに先ほどのAPIキーを貼り付けて保存

```
GEMINI_API_KEY=AIza...（あなたのキー）
```

### 1-4. 語彙リスト（設定済み）

生徒の英文に含まれる「難しい単語」を客観的に判定するための語彙リストです。

- `data/wordlist.csv` に **CEFR-J Wordlist**（約7,000語・A1〜B2レベル）を変換したものを同梱しています。
  - 出典：CEFR-J Wordlist（東京外国語大学 投野由紀夫研究室）。
    Open Language Profiles（Octanove Labs）公開のCSV版をもとに `word,level` 形式へ変換。
  - 変換元データは `data/cefrj-vocabulary-profile-1.5.csv` に、変換スクリプトは
    `scripts/convert-cefrj-wordlist.py` に保存してあります（`python3 scripts/convert-cefrj-wordlist.py` で再生成可能）。
- B1以下の単語を「基準内」、B2以上またはリスト未収載の単語を「高難度」として扱います（活用形は原形に戻して照合）。
- 別のリストを使いたい場合は、同じ `word,level` の2列形式で置き換えてください。

### 1-5. アプリの起動

ターミナルでこのフォルダを開き、次の2つのコマンドを順に実行します。

```bash
npm install    # 初回のみ（部品のダウンロード）
npm run dev    # アプリの起動
```

ブラウザで **http://localhost:3000** を開くと画面が表示されます。終了するときはターミナルで `Ctrl + C` を押します。

## 2. 使い方（Phase 1 時点）

1. **課題を作成**：課題名・出質モード（アイデア重視／論理重視／語彙文法チェック）・出題数を入力して「課題を作成」
2. **英文を投入**：
   - 1人分ずつ：生徒ラベル（出席番号など）と英文を貼り付けて「投入する」
   - 一括：`student_label,text` の2列のCSVファイルを読み込み（または内容を貼り付け）
   - 対象は 30〜300語・3文以上の英文（違反は生成時に警告されます）
3. **生成を実行**：「生成実行へ」→「全員分を生成」（または生徒ごとに「生成」）。
   進捗は生徒ごとにリアルタイム表示されます。無料枠のレート制限対応のため
   **1人あたり1分前後**かかります（40人分で40分〜1時間程度。実行中はブラウザを閉じないでください）。
   Gemini 無料枠は 1日あたりの回数制限もあるため、1日に処理するのは1〜2クラス分までを推奨します
4. **レビュー**：問い・模範解答・採点基準・出題根拠（本文ハイライト）を確認し、
   気に入らない問いは「差し替え」（次点候補と1クリック交換）または「手動編集」。
   問題がなければ「全問を承認する」
5. **印刷**：印刷画面で生徒配布用プリント（1人＝A4縦1枚）と採点キー（教員用）を
   それぞれブラウザの印刷ダイアログから印刷（「PDFとして保存」も可）

> **プライバシーの推奨運用**：生徒ラベルには氏名ではなく**出席番号**を使ってください。

## 3. データの保存場所（ローカルJSON／Googleスプレッドシート）

- 既定では、このフォルダ内の `data/db/*.json` に保存されます（ローカルJSON）。**この設定のままでも全機能が使えます。**
- Google スプレッドシートに保存したい場合（Drive上で提出英文・生成問題・採点結果を直接閲覧できるようになります）は、以下の手順で「サービスアカウント」を作成してください。**新しいアカウント登録は不要**（お持ちの Google アカウントでできます）。**課金設定（クレジットカード）も不要**です。

### サービスアカウントの作成手順（初回のみ・約10分）

1. **Google Cloud Console にログイン**：https://console.cloud.google.com/ を開き、普段の Google アカウントでログイン（初回は利用規約に同意）
2. **プロジェクトを作成**：画面上部のプロジェクト選択 → 「新しいプロジェクト」→ 名前は `oral-exam` など → 作成
3. **Sheets API を有効化**：左メニュー「APIとサービス」→「ライブラリ」→ `Google Sheets API` を検索 → 「有効にする」
4. **サービスアカウントを作成**：「APIとサービス」→「認証情報」→「認証情報を作成」→「サービスアカウント」→ 名前は `oral-exam-bot` など → 「完了」（ロールの付与は不要・スキップでOK）
5. **キー（JSONファイル）を発行**：作成したサービスアカウントをクリック →「キー」タブ →「鍵を追加」→「新しい鍵を作成」→ JSON → 自動でダウンロードされます
6. **キーを配置**：ダウンロードしたJSONファイルをこのフォルダの `secrets/` に移動（例：`secrets/service-account.json`）。このフォルダは Git に含まれません
7. **スプレッドシートを作成して共有**：https://sheets.new で空のスプレッドシートを作成 → 右上「共有」→ サービスアカウントのメールアドレス（手順4で作成した `xxx@xxx.iam.gserviceaccount.com`。キーJSONの `client_email` にも書いてあります）を入力し、**編集者**として共有
8. **`.env.local` に記入**：

```
GOOGLE_SERVICE_ACCOUNT_KEY_PATH=secrets/service-account.json
SPREADSHEET_ID=（スプレッドシートURLの /d/ と /edit の間の長い文字列）
STORAGE_DRIVER=sheets
```

9. アプリを再起動（`Ctrl+C` → `npm run dev`）。初回アクセス時に4つのタブ（assignments / submissions / questions / results）が自動作成されれば成功です。

> 注意：既存のローカルJSONデータは自動では移行されません。切り替えは新しい課題を始めるタイミングがおすすめです。

## 3.5 採点結果の入力と分析（Phase 3）

テスト実施後、「採点・分析へ」画面で各問いに◯/×を入力すると、以下が自動集計されます：

- **正答率（項目困難度）**：型ごとの正答率。0.95超（誰でも解ける）や0.2未満（本人でも解けない疑い）は⚠表示
- **弁別指数**：成績上位群と下位群の正答率差（生徒4人以上で算出）。低い型は「弁別できていない」＝プロンプト改訂の検討対象
- 1問の失点で丸投げと断定せず、複数問・複数回の傾向で判断してください

## 3.6 プロンプト改訂時の回帰テスト（Phase 3）

`prompts/` を変更したら、実クラスに使う前に回帰テストを実行してください：

```bash
npm run golden -- --limit 3   # まず3本で動作確認（数分）
npm run golden                # 全22本（API約150回・30分前後）
```

結果は `data/golden/report-<日時>.md` に保存されます。前回のレポートと Gate1〜3 通過率・型分布を比較し、劣化していれば変更を見直してください。実データ（data/db）には影響しません。

## 4. プロンプトの調整（発展）

- 問題生成のAIへの指示文は `prompts/` フォルダ内のファイルで管理しています（コードには埋め込んでいません）。
- 各ファイルの先頭に `version: vX.Y` が記載されています。**内容を変更したら version を上げてください**（どのバージョンで生成された問題かが記録されます）。
- プロンプトを変更したら、回帰テスト（`npm run golden` → §3.6）を実行してから実クラスに使用してください。

## 4.5 成績評価に使う前に：弁別実験

このテストを成績に反映させる前に、**弁別実験**（自作英文の生徒は解け、AI丸投げの英文では解けないことの実地確認）を一度実施してください。手順・合格基準・結果の読み方は [docs/validation-experiment-guide.md](docs/validation-experiment-guide.md) にまとめてあります。

- 合格基準：**自作群の正答率80%以上**（最優先）・丸投げ群との**正答率差40ポイント以上**
