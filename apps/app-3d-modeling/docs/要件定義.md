# 要件定義：3D Room Viewer（リッチ版）

## 1. プロジェクト概要

### 1.1 目的
高校生がGAS上の簡易3Dモデリングツールで作成した部屋レイアウトを、**リアルで高品質な3Dビュー**として閲覧できるWebページを新規作成する。編集機能は不要で、**View専用ページ**とする。

### 1.2 背景
- GAS上で3Dモデリングの編集・配置は完成済み
- オブジェクトデータはFirebase Realtime Databaseに保存済み
- GASでのレンダリングは解像度が低く、チープな印象を与えてしまう
- 聴衆（発表の場）に対して「高校生が作った」と諦められないクオリティを目指す

### 1.3 デザインの方向性
参考画像の特徴を踏まえ、以下のような**リアル寄りのインテリアビジュアライゼーション**を目指す：
- 木目テクスチャのフローリング床
- 白〜ライトグレーの壁（微細なテクスチャ感）
- 家具には質感のあるマテリアル（布、木、金属など）
- ソフトシャドウとアンビエントオクルージョンによる奥行き感
- 自然光風のライティング（窓からの光の差し込み）

## 3. Firebase データ構造

### 3.1 JSONスキーマ

```
{
  "rooms": {
    "room1": {
      "objects": {
        "<firebase_push_id>": {
          "pos": { "x": number, "y": number, "z": number },
          "rot": { "x": number, "y": number, "z": number },
          "scl": { "x": number, "y": number, "z": number },
          "type": string
        }
      }
    },
    "room2": {
      "objects": { ... }  // 同じ構造
    }
  }
}
```

### 3.2 座標系の仕様
- **pos**: オブジェクトの位置（x, y, z）。yは上方向（高さ）
- **rot**: オブジェクトの回転（ラジアン）。Three.jsのEuler角に直接対応
- **scl**: オブジェクトのスケール（倍率）
- 座標値に制限はない（配置可能範囲外のオブジェクトも存在する）

### 3.3 オブジェクトタイプ一覧

以下の10種類のオブジェクトタイプが存在する：

| タイプ | Room1個数 | Room2個数 | 説明 |
|--------|-----------|-----------|------|
| `wall` | 49 | 25 | 壁パネル。scl.xで幅を調整している場合あり |
| `table` | 30 | 4 | テーブル・デスク |
| `door` | 17 | 12 | ドア。開閉状態は不問 |
| `floor` | 14 | 0 | 床パネル（Room2には存在しない） |
| `plant` | 11 | 0 | 観葉植物 |
| `chair` | 6 | 6 | 椅子 |
| `window` | 4 | 0 | 窓 |
| `stairs` | 2 | 2 | 階段 |
| `elevator` | 2 | 3 | エレベーター |
| `sofa` | 0 | 10 | ソファ（Room2のみ） |

### 3.4 各ルームの座標範囲

**Room 1:**
- 壁のX範囲: -9.25 〜 9
- 壁のZ範囲: -8.75 〜 18
- 全オブジェクトのX範囲: -15 〜 18.25（壁の範囲外にテーブル・植物・椅子あり）
- 全オブジェクトのZ範囲: -278 〜 19.5（floorオブジェクトが遠方に存在）

**Room 2:**
- 壁のX範囲: -8.25 〜 8.75
- 壁のZ範囲: -6.5 〜 8.25
- 全オブジェクトのX範囲: -8.75 〜 9.5
- 全オブジェクトのZ範囲: -10.25 〜 12.75

### 3.5 データに関する重要な注意事項

#### 配置可能範囲外のオブジェクト
GASの編集ツールでは配置可能範囲が設定されていたが、一部のオブジェクトはその範囲外に配置されている。**これらは意図的な配置であり、すべて正確に反映すること。** 具体例：
- Room1: X=15〜18付近にテーブル群・植物群・椅子（壁の右外側）
- Room1: X=-15付近にテーブル群・椅子（壁の左外側）
- Room1: Z=-278, Z=-28.75にfloorオブジェクト（壁の遠方）
- Room1: Z=19.5にテーブル（壁のやや外）

#### 異常値に見えるスケール
Room2のchairオブジェクト1つに `scl: (1553.13, -204185.87, 1.78)` という極端なスケール値がある。ユーザーは意図的と回答しているが、**この値をそのまま Three.js に適用するとレンダリングに悪影響を及ぼす可能性が極めて高い。** 以下の対策を実装すること：
- スケール値にサニタイズ処理を入れる（例：各軸 -100〜100 にクランプ）
- クランプが適用されたオブジェクトはコンソールに警告を出力する
- サニタイズの閾値は設定ファイルで変更可能にする

## 5. 3Dレンダリング仕様

### 5.1 各オブジェクトタイプの3Dモデル定義

Firebaseから取得したデータに基づき、各 `type` に対応する3Dジオメトリとマテリアルを生成する。**GLBモデルの外部読み込みは行わず、Three.jsのプリミティブジオメトリを組み合わせてプロシージャルに生成する。** ただし、テクスチャ画像は使用してよい。

#### wall（壁）
- ジオメトリ: BoxGeometry（薄い直方体）
- ベースサイズ: 幅2 × 高さ1.5 × 奥行き0.1（sclで調整される）
- マテリアル: MeshStandardMaterial
  - 色: 白〜オフホワイト (#F5F5F0)
  - roughness: 0.9
  - 微細な凹凸テクスチャ（normalMap）を適用して塗り壁感を出す

#### floor（床）
- ジオメトリ: BoxGeometry（平たい直方体）
- ベースサイズ: 幅2 × 高さ0.05 × 奥行き2
- マテリアル: MeshStandardMaterial
  - 木目テクスチャ（フローリング調）
  - roughness: 0.6
  - 明るいオーク調の色味

#### table（テーブル）
- プロシージャルモデル: 天板（BoxGeometry）+ 脚4本（CylinderGeometry）
- ベースサイズ: 天板 幅1.2 × 高さ0.05 × 奥行き0.6、脚は天板の四隅
- マテリアル: MeshStandardMaterial
  - 天板: 木目テクスチャ（ウォールナット調）
  - 脚: やや暗めの木目 or 金属

#### chair（椅子）
- プロシージャルモデル: 座面（BoxGeometry）+ 背もたれ（BoxGeometry）+ 脚4本
- ベースサイズ: 座面 幅0.5 × 高さ0.05 × 奥行き0.5、全体の高さ約0.8
- マテリアル:
  - 座面・背もたれ: ファブリック調（グレー系）、roughness: 0.8
  - 脚: メタル調、metalness: 0.6

#### door（ドア）
- プロシージャルモデル: ドアパネル（BoxGeometry）+ フレーム（BoxGeometryで枠）
- ベースサイズ: 幅1 × 高さ2 × 奥行き0.08
- マテリアル:
  - パネル: 木目テクスチャ（明るめ）
  - フレーム: 白 or ライトグレー
  - ドアノブ: 小さな球体（金属マテリアル）

#### window（窓）
- プロシージャルモデル: フレーム（BoxGeometryで枠）+ ガラス面（PlaneGeometry）
- ベースサイズ: 幅1.2 × 高さ1.0 × 奥行き0.08
- マテリアル:
  - フレーム: 白 or アルミ色
  - ガラス: MeshPhysicalMaterial（transmission: 0.9, roughness: 0.05, transparent）
  - 窓の外は明るい空色（環境光源として機能させる）

#### plant（観葉植物）
- プロシージャルモデル: 鉢（CylinderGeometry）+ 葉のクラスター（SphereGeometryまたはcone群）
- ベースサイズ: 全体の高さ約0.8
- マテリアル:
  - 鉢: テラコッタ調（茶色、roughness: 0.8）
  - 葉: グリーン系（#3A7D44程度）、roughness: 0.7
  - 複数の球体を重ねて葉のボリューム感を出す

#### sofa（ソファ）
- プロシージャルモデル: 座面 + 背もたれ + 肘掛け（すべてBoxGeometryの角丸的表現）
- ベースサイズ: 幅2 × 高さ0.8 × 奥行き0.9
- マテリアル:
  - ファブリック調（ダークグレー or ベージュ）、roughness: 0.85
  - クッション部分は少し明るめの色でアクセント

#### stairs（階段）
- プロシージャルモデル: 段を複数のBoxGeometryで積み上げ（6〜8段）
- ベースサイズ: 幅1.5 × 高さ1.5 × 奥行き2
- マテリアル:
  - 木目テクスチャ（床と同系色）
  - 手すり: CylinderGeometry × 金属マテリアル

#### elevator（エレベーター）
- プロシージャルモデル: 箱型の筐体（BoxGeometry）+ ドア（中央分割の2枚パネル）
- ベースサイズ: 幅1.5 × 高さ2.2 × 奥行き1.5
- マテリアル:
  - 筐体: ステンレス調（metalness: 0.7, roughness: 0.3）
  - ドア: やや暗めのメタル
  - ドア横にボタンパネル（小さなPlane）

### 5.2 自動生成する床（グラウンドプレーン）

Firebaseデータの `floor` オブジェクトとは別に、**部屋全体をカバーする大きな床面**を自動生成する。

- 各ルームの壁オブジェクトの座標範囲から自動計算
- 壁範囲より各方向に2ユニットの余白を取る
- Room1推定サイズ: 約 X: -11〜11, Z: -11〜20（壁範囲 + マージン）
- Room2推定サイズ: 約 X: -10〜11, Z: -8〜10（壁範囲 + マージン）
- マテリアル: 木目フローリング（参考画像と同様のオーク調）
- 配置可能範囲外にオブジェクトがある場合はそのオブジェクトもカバーするように床を拡張する

### 5.3 ライティング

リアル感を出すために複数の光源を組み合わせる：

| 光源 | 種類 | 設定 |
|------|------|------|
| メインライト | DirectionalLight | 太陽光的な斜め上からの照射。castShadow: true |
| フィルライト | DirectionalLight | メインの反対側から弱い光。影なし |
| 環境光 | AmbientLight | 全体を均一に明るくする補助光。intensity: 0.3程度 |
| 環境マップ | drei の Environment | HDR環境マップで自然な反射と間接光 |

### 5.4 シャドウ

- DirectionalLight に `castShadow: true`
- shadow mapサイズ: 2048 × 2048
- 床とオブジェクトに `receiveShadow: true`
- 家具オブジェクトに `castShadow: true`
- ソフトシャドウを実現するため `shadow.radius` を設定

### 5.5 ポストプロセッシング（後処理エフェクト）

`@react-three/postprocessing` を使って以下を適用：
- **SSAO（Screen Space Ambient Occlusion）**: 隅や接地面に自然な陰影
- **Bloom（控えめ）**: 窓からの光に微かな輝き感
- **ToneMapping**: ACESFilmic で映画的な色味

### 5.6 パフォーマンス配慮

- Room1のオブジェクト数: 135個、Room2: 62個（合計約200個）
- 同時に表示するのは1ルーム分のみ（ルーム切り替え時にシーンを入れ替え）
- ジオメトリのインスタンシングを活用（同じtypeのオブジェクトはInstancedMeshで描画）
- テクスチャは再利用（同じマテリアルを共有）
- LOD（Level of Detail）は今回不要（オブジェクト数が少ないため）

## 6. データフロー

```
Firebase Realtime Database
        │
        ▼
  Next.js クライアント（useEffect + onValue）
        │
        ▼
  データパース & サニタイズ
  （異常スケール値のクランプ処理）
        │
        ▼
  ルーム選択状態に応じてフィルタリング
        │
        ▼
  React Three Fiber シーンにオブジェクトをマッピング
        │
        ▼
  各 type に対応するコンポーネントで3Dモデルを生成
```

### 6.1 データサニタイズ処理

```typescript
// 設定値（config.tsで管理）
const SCALE_CLAMP = {
  min: -100,
  max: 100,
};

function sanitizeScale(scl: Vec3): Vec3 {
  const clamp = (v: number) =>
    Math.max(SCALE_CLAMP.min, Math.min(SCALE_CLAMP.max, v));
  
  const sanitized = {
    x: clamp(scl.x),
    y: clamp(scl.y),
    z: clamp(scl.z),
  };
  
  if (sanitized.x !== scl.x || sanitized.y !== scl.y || sanitized.z !== scl.z) {
    console.warn('スケール値がクランプされました:', { original: scl, sanitized });
  }
  
  return sanitized;
}
```

## 11. 実装上の注意点

### 11.1 配置可能範囲外オブジェクトの扱い
**最重要事項**: Firebaseから取得した全オブジェクトを、座標値に関わらず忠実にレンダリングすること。フィルタリングや除外は一切行わない。自動生成する床面（GroundPlane）は、全オブジェクトをカバーするように動的にサイズを計算する。

### 11.2 スケール値の扱い
- 各オブジェクトの `scl` をThree.jsの `scale` プロパティに直接適用する
- ただし、事前にサニタイズ（クランプ）処理を通す
- クランプ閾値はデフォルト -100〜100 とし、`config.ts` で変更可能

### 11.3 回転値の扱い
- `rot` の値はラジアン単位で格納されている
- Three.jsの `rotation` プロパティ（Euler角）にそのまま適用可能
- 一部のオブジェクトで `rot.x` や `rot.z` に π（3.14...）が入っている = 180度回転

### 11.4 テクスチャの用意
テクスチャ画像は以下の方法で用意する（優先順）：
1. フリーのテクスチャサイト（Poly Haven, ambientCGなど）からダウンロードしてpublic/textures/に配置
2. プロシージャルテクスチャ（コードで生成）
3. drei の `useTexture` でロード

## 14. 開発の進め方（推奨順序）

1. **プロジェクト初期化**: Next.js + TypeScript + Tailwind CSS のセットアップ
2. **Firebase接続**: データ取得 & 型定義 & サニタイズ処理
3. **基本シーン構築**: React Three Fiber のCanvas、ライティング、OrbitControls
4. **オブジェクトコンポーネント作成**: wall → floor → table の順に、シンプルなBoxGeometryから開始
5. **全オブジェクトタイプの実装**: 残りのタイプを順次実装
6. **マテリアル & テクスチャ適用**: 木目、ファブリック、金属などのリッチなマテリアル
7. **GroundPlane自動生成**: 壁の座標範囲から床面を動的に生成
8. **ルーム切り替えUI**: Room1/Room2の切り替え機能
9. **ポストプロセッシング**: SSAO、Bloomの追加
10. **UI仕上げ**: ローディング画面、ヘルプオーバーレイ
11. **パフォーマンス最適化**: InstancedMesh化、テクスチャ圧縮
12. **Vercelデプロイ**: 環境変数設定、本番ビルド確認
